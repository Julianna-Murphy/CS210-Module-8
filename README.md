# CS210-Module-8 - Julianna Murphy

## Summarize the project and what problem it was solving.
Project 2, the Airgead Banking Calculator, is a C++ console-based investment calculator that uses user input to calculate compound interest. This project shows the significance of compound interest on a user's investment and how much it can be impacted whether the user continues to add money or if no monthly installments are made.

## What did you do particularly well?
I was particularly well in creating a user friendly experience that is condensed into less than 200 lines of code. I felt pretty confident in the comments I made as well to help the user understand the code step by step.

## Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
I definitely could have enhanced my code by creating a separate menu for the user input values (initial investment, monthly deposit, annual interest and number of years) and their absolute value. I had originally put it in main, but I could have condensed main slightly by putting the user input values as parameters into a function and calculate the absolute value in another function.

## Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
I found the most challenging part to be getting the UI to be condensed but also readable. I tried various sizing options until I found one that felt 'right'. I found tremendous value in learning C++ and can add that to the other languages I know (Java, Python).

## What skills from this project will be particularly transferable to other projects or course work?
There are various skill that can be transferable to other projects. C++ is a widely used object oriented programming language by many companies. Input validation is also a very important skill in every day projects and can be translated to any programming language.

## How did you make this program maintainable, readable, and adaptable?
This program is maintainable and adaptable by creating modular functions that can be easily transferred to another project. The program has less than 200 lines which can help navigating through less code. The naming convention used in the program is also helpful when understanding the variables and methods/functions used in the program as well as following the comments to understand the flow of the program.
